# 项目推进监督角色章程

创建日期：2026-04-30  
适用项目：AD 外周血转录组诊断标志物纯生信项目  
目标：确保项目最后能落地为可投稿、可复现、可审核的 SCI Q3/Q4 稿件。

## 一、监督原则

1. 监督角色只做审核、阻断、建议和放行，不替代实际分析。
2. 所有放行必须基于证据文件，不基于口头感觉。
3. P0 问题未关闭时，不允许进入下一关键阶段。
4. 审稿模拟必须保留反对意见，不能只给正面判断。
5. 投稿前必须完成完整性核查、复现核查和目标期刊核查。

## 二、角色配置

| 角色 | 职责 | 审核时点 | 是否可阻断 |
|---|---|---|---|
| Pipeline Lead | 维护总计划、日志、阶段推进和交付物 | 每周 | 是 |
| Data Integrity Auditor | 核查数据来源、样本标签、纳入排除、下载记录 | S1、S2、S8 | 是 |
| Methodology Reviewer | 核查差异分析、建模、防泄漏、统计报告 | S3、S5、S8 | 是 |
| Reproducibility Auditor | 核查脚本、环境、目录、结果能否重跑 | S2 起每阶段 | 是 |
| Domain Reviewer | 核查 AD、生物学解释、外周血合理性 | S4、S7、S8 | 是 |
| Devil's Advocate | 专门寻找最强反对意见和 desk reject 风险 | S0、S5、S8 | 是 |
| Journal Fit Strategist | 核查目标期刊范围、JCR/SCIE 状态、APC 和风险 | S7、S9 | 是 |
| Writing Quality Reviewer | 核查论文结构、图表叙事、语言、声明和引用 | S7、S8 | 条件阻断 |

## 三、阶段放行门槛

### Gate 0：立项放行

状态：已条件通过。  
必须满足：
- 有可行性核查报告。
- 有项目日志。
- 有总推进计划。
- 有监督角色章程。

### Gate 1：数据冻结放行

必须满足：
- GSE63060、GSE63061、GSE85426 下载记录齐全。
- 每个文件有下载日期、大小、来源链接。
- 样本标签表完成并可追溯。
- 明确排除 MCI 和标签不清样本。

阻断条件：
- 样本标签无法复核。
- AD/Control 子集与文献或 GEO metadata 不一致且无解释。

### Gate 2：预处理与 QC 放行

必须满足：
- probe 到 gene symbol 的映射规则明确。
- 重复 probe 折叠规则明确。
- PCA、箱线图或 RLE 显示无明显异常。
- 排除样本有记录。

阻断条件：
- 预处理后样本或基因数量无法解释。
- 平台差异被错误当作疾病信号。

### Gate 3：候选基因与功能解释放行

必须满足：
- DEG 阈值、候选基因筛选阈值写入参数表。
- 富集分析使用 BH 或等价多重校正。
- 至少 3 个核心通路能和 AD/免疫/炎症/神经退行逻辑连接。

阻断条件：
- 只报显著条目，不解释生物学意义。
- 候选基因数量过多导致建模不可解释。

### Gate 4：建模与外部验证放行

必须满足：
- 特征选择和调参只发生在训练集内部。
- 外部验证集只做最终评估。
- 至少报告 AUC、95% CI、敏感度、特异度。
- 最终模型特征数控制在 3 到 8 个。
- 至少一个外部验证集 AUC >= 0.75。

阻断条件：
- 发现任何训练/验证泄漏。
- 两个外部验证集表现均低于可解释阈值。
- 外部验证方向与训练集不一致。

### Gate 5：稿件初稿放行

必须满足：
- Methods 能完整复现流程。
- Results 每张图都有源数据。
- Discussion 不夸大因果。
- Limitations 明确说明公共数据、无湿实验、跨平台差异。
- Data Availability、Ethics、Author Contributions、Conflict of Interest、Funding、AI disclosure 齐全。

阻断条件：
- 引用不实或来源无法验证。
- 结论超过数据支持范围。

### Gate 6：预投稿放行

必须满足：
- 引用与参考文献 100% 匹配。
- 关键数据和图表可重跑。
- 候选期刊完成当年 SCIE/JCR 分区复核。
- 审稿模拟完成并关闭 P0/P1 问题。

阻断条件：
- 目标期刊 SCIE 状态无法确认。
- 期刊范围与稿件明显不匹配。
- 存在未关闭的 P0/P1 方法或完整性问题。

## 四、审核输出格式

每次审核输出必须使用下列格式：

```text
审核日期：
审核角色：
审核阶段：
输入材料：
结论：PASS / CONDITIONAL PASS / FAIL
P0 问题：
P1 问题：
P2 建议：
证据：
放行条件：
下一次审核时点：
```

## 五、项目是否能落地的最终判断标准

满足以下全部条件，才判定“可以落地投稿”：

1. 三个 GEO 数据集可追溯，样本表无硬伤。
2. 主流程一键或半自动重跑成功。
3. 至少一个外部验证集表现达到预设门槛。
4. 最终基因具有合理生物学解释。
5. 所有图表有源数据和脚本。
6. 稿件完成完整性审核和审稿模拟。
7. 目标期刊确认 SCIE 收录、分区、范围和 APC。

若只满足 1 到 5，不进入投稿，只能进入“补写与补审”。
